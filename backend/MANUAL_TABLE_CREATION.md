# Manual Payment Configs Table Creation

## ⚠️ IMPORTANT: MANUAL SETUP REQUIRED

The `payment_configs` table needs to be created manually via the Supabase dashboard because direct SQL execution is not available through the API.

## 📋 STEPS TO CREATE TABLE

### 1. Go to Supabase Dashboard
- Navigate to: https://slewmhaflrplgedgfvmz.supabase.co
- Go to **SQL Editor** tab

### 2. Execute the Following SQL

```sql
-- Create the payment_configs table
CREATE TABLE IF NOT EXISTS public.payment_configs (
  config_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  school_id BIGINT NOT NULL REFERENCES public.schools(school_id) ON DELETE CASCADE,
  
  -- M-Pesa Configuration
  mpesa_till_number VARCHAR(20) NULL,
  mpesa_paybill VARCHAR(20) NULL,
  mpesa_shortcode VARCHAR(20) NULL,
  mpesa_consumer_key TEXT NULL,
  mpesa_consumer_secret TEXT NULL,
  mpesa_passkey TEXT NULL,
  mpesa_environment VARCHAR(20) NOT NULL DEFAULT 'sandbox',
  mpesa_base_url VARCHAR(255) NULL,
  mpesa_callback_base_url VARCHAR(255) NULL,
  
  -- Paystack Configuration
  paystack_public_key TEXT NULL,
  paystack_secret_key TEXT NULL,
  paystack_environment VARCHAR(20) NOT NULL DEFAULT 'test',
  paystack_callback_url VARCHAR(255) NULL,
  
  -- Bank Configuration
  bank_account_number VARCHAR(50) NULL,
  bank_name VARCHAR(100) NULL,
  bank_branch VARCHAR(100) NULL,
  account_name VARCHAR(200) NULL,
  
  -- SMS Configuration
  sms_enabled BOOLEAN NOT NULL DEFAULT false,
  sms_sender_id VARCHAR(20) NULL,
  
  -- Metadata
  is_active BOOLEAN NOT NULL DEFAULT true,
  is_deleted BOOLEAN NOT NULL DEFAULT false,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Create indexes for performance
CREATE INDEX IF NOT EXISTS idx_payment_configs_school_id ON public.payment_configs(school_id);
CREATE INDEX IF NOT EXISTS idx_payment_configs_active ON public.payment_configs(is_active, is_deleted);

-- Partial unique index: Ensure each school has only one active config
CREATE UNIQUE INDEX IF NOT EXISTS idx_payment_configs_unique_active 
ON public.payment_configs(school_id) 
WHERE (is_active = true AND is_deleted = false);

-- Enable Row Level Security (RLS)
ALTER TABLE public.payment_configs ENABLE ROW LEVEL SECURITY;

-- Create RLS Policies (drop if exists first, then create)
DROP POLICY IF EXISTS "Schools can view own payment configs" ON public.payment_configs;
CREATE POLICY "Schools can view own payment configs" ON public.payment_configs
  FOR SELECT USING (school_id = (auth.jwt() ->> 'school_id')::bigint);
  
DROP POLICY IF EXISTS "Schools can insert own payment configs" ON public.payment_configs;
CREATE POLICY "Schools can insert own payment configs" ON public.payment_configs  
  FOR INSERT WITH CHECK (school_id = (auth.jwt() ->> 'school_id')::bigint);
  
DROP POLICY IF EXISTS "Schools can update own payment configs" ON public.payment_configs;
CREATE POLICY "Schools can update own payment configs" ON public.payment_configs
  FOR UPDATE USING (school_id = (auth.jwt() ->> 'school_id')::bigint);
```

### 3. Verify Table Creation
After running the SQL, verify the table was created by running:
```sql
SELECT COUNT(*) FROM public.payment_configs;
```
